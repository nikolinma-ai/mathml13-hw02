## Основные понятия

### Стационарность

**Стационарный ряд** — ряд, у которого статистические свойства не меняются во времени. Это значит:

- среднее значение постоянно — ряд не «уплывает» вверх или вниз;
- дисперсия постоянна — амплитуда колебаний не растёт со временем;
- автокорреляция между точками зависит только от расстояния между ними, а не от их положения на оси времени.

Проще: если взять любой кусок ряда — он статистически выглядит так же, как любой другой кусок.

Зачем это нужно: классические модели (AR, MA, ARMA) построены в предположении стационарности. Если ряд нестационарен, модель будет ловить «привидения» — ложные зависимости, которые не воспроизведутся на новых данных.

### Тест Дики-Фуллера (ADF)

**ADF-тест** проверяет гипотезу о наличии единичного корня — это формальный признак нестационарности.

- **H0** (нулевая гипотеза): ряд **нестационарен** (есть единичный корень).
- **H1**: ряд стационарен.

Решение принимается по двум числам: **статистике теста** и **p-value**.

Правила:

- если p-value < 0.05 — H0 отвергается, ряд считается стационарным;
- если статистика **меньше** критического значения на нужном уровне (например, на 5%) — H0 тоже отвергается.

Чем «более отрицательная» статистика — тем увереннее отвергаем нестационарность.

### Дифференцирование

**Дифференцирование** — простой и мощный приём, чтобы убрать тренд.

Идея: вместо самого ряда работаем с его **разностями**:

```
y'(t) = y(t) - y(t-1)
```

После этого тренд исчезает (в среднем разности болтаются вокруг нуля), и ряд часто становится стационарным.

Если первая разность всё ещё нестационарна — берут вторую разность (разность от разности). Параметр **d** в ARIMA обозначает именно порядок дифференцирования. Обычно d=0, 1 или 2; больше двух — плохой знак.

Полезный приём: для рядов с экспоненциальным ростом (ВВП, выручка стартапа) сначала берут **логарифм**, а потом дифференцируют. Логарифм стабилизирует дисперсию, дифференцирование убирает тренд.

### Декомпозиция

**Декомпозиция** — разложение ряда на три компоненты:

- тренд **T(t)**;
- сезонность **S(t)**;
- остаток (шум) **R(t)**.

Две схемы:

- **Аддитивная**: y(t) = T(t) + S(t) + R(t). Подходит, когда амплитуда сезонности **постоянна**.
- **Мультипликативная**: y(t) = T(t) · S(t) · R(t). Подходит, когда амплитуда сезонности **растёт** вместе с уровнем ряда.

В Python — `seasonal_decompose` из statsmodels. Обязательно нужен параметр **period** — длина сезонного цикла в наблюдениях.

### Автокорреляция: ACF и PACF

**ACF (Autocorrelation Function)** — корреляция ряда со своими лагами. ACF(k) показывает, насколько y(t) связан с y(t-k).

**PACF (Partial Autocorrelation Function)** — то же, но «очищенное» от влияния промежуточных лагов. PACF(k) показывает прямую связь y(t) с y(t-k), без участия y(t-1), …, y(t-k+1).

Эти две функции — главный инструмент для **выбора порядка** модели ARIMA на глаз:

- по PACF — определяют **p** (порядок AR-части): на каком лаге PACF резко обрывается.
- по ACF — определяют **q** (порядок MA-части): на каком лаге ACF резко обрывается.

### Модель ARIMA

**ARIMA(p, d, q)** — модель, которая объединяет три механизма:

- **AR(p)** — авторегрессия: текущее значение зависит от p предыдущих значений ряда;
- **I(d)** — интегрирование: предварительное d-кратное дифференцирование, чтобы ряд стал стационарным;
- **MA(q)** — скользящее среднее ошибок: текущее значение зависит от q предыдущих ошибок прогноза.

Параметры:

- **p** — порядок AR-части (типично 0–3);
- **d** — порядок дифференцирования (0, 1, 2);
- **q** — порядок MA-части (типично 0–3).

Способы выбора (p, d, q):

1. **d** — экспериментально через ADF: дифференцируй ряд, пока ADF не скажет «стационарен».
2. **p, q** — по графикам PACF и ACF либо перебором по информационному критерию AIC.

### Модель SARIMA

**SARIMA(p, d, q)(P, D, Q, s)** — расширение ARIMA с сезонными компонентами:

- **P, D, Q** — сезонные порядки AR, I, MA;
- **s** — длина сезонного цикла (12 для месячных данных с годовой сезонностью, 7 для дневных с недельной).

SARIMA нужна, когда ряд имеет явную сезонность — продажи мороженого, потребление электричества, посещаемость сайта.

### Walk-forward валидация

Кросс-валидация k-fold для временных рядов **не работает** — она перемешивает наблюдения и нарушает причинность.

**Walk-forward (rolling origin)** — правильный способ:

1. Берёшь первый кусок истории — это train.
2. Прогнозируешь следующие h точек — это test.
3. Сдвигаешь окно вперёд, дообучаешь модель.
4. Повторяешь много раз. Усредняешь ошибки.

В sklearn это `TimeSeriesSplit`. Главный принцип: **тестовая выборка всегда после тренировочной**.

### GARCH — модель волатильности

ARIMA моделирует **средний** уровень ряда. Но в финансах часто важна не средняя цена, а **разброс** — волатильность.

**GARCH(p, q)** моделирует условную дисперсию: текущая волатильность зависит от прошлых волатильностей и прошлых шоков. Это стандартный инструмент для риск-менеджмента — расчёта VaR, маржи, цен опционов.

В Python — библиотека `arch`, класс `arch_model`.

### Информационные критерии

Когда параметров много, на глаз выбрать тяжело. На помощь приходят **информационные критерии**:

- **AIC** (Akaike Information Criterion);
- **BIC** (Bayesian Information Criterion).

Логика одинаковая: чем меньше — тем лучше. Они балансируют качество подгонки и сложность модели (число параметров). Перебираешь сетку (p, d, q) → выбираешь модель с минимальным AIC.

## Пошаговый разбор

### Шаг 1. Проверь стационарность

```python
from statsmodels.tsa.stattools import adfuller

result = adfuller(series)
print('ADF Statistic:', result[0])
print('p-value:', result[1])
print('Critical values:', result[4])
```

Если p-value > 0.05 — ряд нестационарен. Идём дифференцировать.

### Шаг 2. Дифференцируй до стационарности

```python
diff1 = series.diff().dropna()
adfuller(diff1)
```

Если по первой разности p-value < 0.05 — d=1. Если нет — берём вторую разность, d=2.

Если ряд растёт по экспоненте — сначала логарифмируй, потом дифференцируй:

```python
import numpy as np
log_series = np.log(series)
log_diff = log_series.diff().dropna()
```

### Шаг 3. Декомпозируй ряд

```python
from statsmodels.tsa.seasonal import seasonal_decompose
decomp = seasonal_decompose(series, model='additive', period=12)
decomp.plot()
```

Смотришь глазами: есть ли явный тренд, есть ли регулярная сезонность.

### Шаг 4. Подбери порядок ARIMA

Подход 1 — графический:

```python
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
plot_pacf(diff_series, lags=20)  # → подсказывает p
plot_acf(diff_series, lags=20)   # → подсказывает q
```

Подход 2 — перебор по AIC:

```python
from statsmodels.tsa.arima.model import ARIMA

best = None
for p in range(3):
    for q in range(3):
        try:
            m = ARIMA(series, order=(p, 1, q)).fit()
            if best is None or m.aic < best[1]:
                best = ((p, 1, q), m.aic)
        except Exception:
            pass
print('best order:', best)
```

### Шаг 5. Подгони модель и сделай прогноз

```python
model = ARIMA(series, order=(1, 1, 1))
fit = model.fit()
forecast = fit.forecast(steps=5)
print(forecast)
```

### Шаг 6. Проверь честно

```python
from sklearn.model_selection import TimeSeriesSplit
from sklearn.metrics import mean_squared_error

tscv = TimeSeriesSplit(n_splits=5, test_size=3)
errs = []
for tr_idx, te_idx in tscv.split(series):
    train = series.iloc[tr_idx]
    test = series.iloc[te_idx]
    fit = ARIMA(train, order=(1, 1, 1)).fit()
    pred = fit.forecast(steps=len(test))
    errs.append(mean_squared_error(test, pred, squared=False))
print('mean RMSE:', np.mean(errs))
```

### Шаг 7. Если есть сезонность — SARIMA

```python
from statsmodels.tsa.statespace.sarimax import SARIMAX
model = SARIMAX(series, order=(1, 1, 1), seasonal_order=(1, 1, 1, 12))
fit = model.fit(disp=False)
```

### Шаг 8. Если важна волатильность — GARCH

```python
from arch import arch_model
returns = np.log(series).diff().dropna() * 100  # в процентах
am = arch_model(returns, vol='Garch', p=1, q=1)
res = am.fit(disp='off')
forecast_var = res.forecast(horizon=5).variance.values[-1]
```

## Примеры

### Пример 1. Месячная выручка кофейни

Дано: ряд месячной выручки за 5 лет, явный рост и пик каждый декабрь.

Действия:

1. ADF — нестационарен.
2. Логарифм + первая разность — стационарен.
3. seasonal_decompose(period=12) — тренд + годовая сезонность.
4. Используем SARIMA(1,1,1)(1,1,1,12).
5. Walk-forward на последние 12 месяцев — RMSE 8% от среднего.

### Пример 2. Прогноз погасания энергопотребления (часовой)

Дано: ряд потребления каждый час за год.

Действия:

1. Видна суточная сезонность (s=24) и недельная (s=168).
2. Берут первую разность сезонную — `series.diff(24)`.
3. SARIMA(1,0,1)(1,1,1,24).
4. Прогноз на следующий день — RMSE сравнивают с baseline «то же время вчера».

### Пример 3. Волатильность акций

Дано: дневные доходности по акции за 3 года.

Действия:

1. ADF — доходности стационарны.
2. ARMA на средние не даёт ничего, остатки **гетероскедастичны**.
3. GARCH(1,1) на квадраты остатков — даёт хорошие оценки волатильности.
4. Прогноз дисперсии используют для расчёта VaR на следующий день.

## Итог

ARIMA и SARIMA — рабочая лошадка прогноза временных рядов. Чтобы их правильно применить, нужен ритуал:

1. Посмотрел на ряд глазами.
2. Проверил стационарность (ADF).
3. Привёл к стационарности (log + diff).
4. Декомпозировал.
5. Подобрал (p, d, q) — по ACF/PACF или по AIC.
6. Если есть сезонность — SARIMA.
7. Валидировал walk-forward.
8. Если важна волатильность — поверх ARIMA натянул GARCH.

Этот ритуал работает на 80% задач прогноза в индустрии. Для оставшихся 20% — Prophet, нейросети, бустинги по временным признакам — но это уже следующие модули.

## Частые ошибки

- **Применил ARIMA к нестационарному ряду без дифференцирования.** Модель сойдётся, прогноз даст, но он будет ложным. Всегда сначала ADF.
- **Перепутал d=2 с d=1.** Если первая разность уже стационарна — d=2 даст переобучение и шумные прогнозы.
- **Использовал k-fold вместо walk-forward.** Метрика будет красивой, но в проде сломается. Время — неперемешиваемо.
- **Смотрел только на AIC, не глядя на остатки.** Хорошая модель должна давать **белый шум** в остатках. Проверка — ACF остатков, тест Льюнга-Бокса.
- **Не учёл сезонность.** На месячных данных без сезонной компоненты ARIMA проиграет SARIMA в разы.
- **Считал GARCH на самих ценах вместо доходностей.** GARCH моделирует условную дисперсию доходностей, а не уровней.
- **Дал SARIMA маленький s.** Если данных меньше двух полных сезонных циклов — сезонная часть SARIMA не выучится.

## Запомни

- ARIMA = AR(p) + I(d) + MA(q). Три буквы, три механизма.
- SARIMA = ARIMA + сезонная часть (P, D, Q, s).
- d — это сколько раз дифференцировать. Обычно 0, 1 или 2.
- ADF: p-value < 0.05 → ряд стационарен.
- p — по PACF, q — по ACF. Или перебор по AIC.
- TimeSeriesSplit, не KFold. Никогда не перемешивай ряд.
- GARCH — для волатильности, не для среднего.
- Логарифмируй, если ряд растёт мультипликативно.
- Чем меньше AIC — тем лучше.
- Остатки модели должны быть белым шумом.
